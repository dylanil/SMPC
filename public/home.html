Secure Multi-Party Average

Three to ten participants each privately enter one figure on their own page. A separate aggregator computes the average from masked shares — raw figures are never transmitted. Rounds can be named for whatever's being benchmarked: e.g. competing insurers privately computing the market's average claim severity.
How to start a round

1. The aggregator opens their page, picks how many participants will join (between 3 and 10), and clicks Create session — the server mints a unique 6-character session code plus one per-role invite token.

2. The aggregator shares each invite (formatted as SESSION-TOKEN) with its matching participant out-of-band (Slack, email, a phone call). Codes are role-bound: a participant given A's invite cannot claim slot B.

3. Each participant opens their page (e.g. /party/A), enters the invite, and submits their figure. The aggregator sees the average once everyone has submitted; each participant page also independently re-verifies the result.

How it works

1. Each pair of participants (i, j) with i < j derives a one-time-pad mask r_ij via ECDH + HKDF. Participant i adds it; participant j subtracts it.

2. Each participant submits only s_i = x_i ± masks — a single number indistinguishable from uniform random.

3. The aggregator sums all N masked shares. Every mask appears once with + and once with −, so the masks cancel and the sum reveals Σ x_i, from which the average follows.
