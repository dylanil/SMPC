Catch a dishonest aggregator - simulation only

Every result can be re-checked: the server returns each masked share, the participant's signature over it, and the verifying key registered at join. In a real round each participant's own page re-runs these checks (party page, Step 6). Here you can watch them catch a forged result.
The tampering above happened to a local copy in this tab only - the round's real data on the server is untouched. A dishonest aggregator can report a self-consistent sum, but it cannot forge a signature over a value the participant never sent - that would need the participant's private signing key, which never left their browser. (Tampered data can't even land server-side: signatures are verified at write time and first-write-wins locks each share.) This check protects result integrity; it does not by itself prove who is behind each party - see the impersonation limitation in the README.
About this demo

This is an educational proof-of-concept, not an audited or production service. It shows the cryptographic plumbing of secure multi-party computation, but it does not verify whether inputs are truthful or reasonable; everyone must finish the round; invite tokens are not real-world identity. Full threat model: README · SECURITY.

Demo provided as-is; do not rely on it for real, confidential, regulated, or production use.

No accounts or analytics. Logs are minimal and ephemeral; IPs are used for rate limiting. Functional auth cookies are used only when password protection is enabled.
