# SMPC ‚Äî Secure Average of Three Private Claims

A small demo of Secure Multi-Party Computation. Three insurers each enter one private number ("claim") from their own browser. A separate aggregator computes the average **without ever seeing any insurer's raw claim**. Security comes from *pairwise one-time-pad masking*: every pair of insurers shares a random mask that cancels out when all three masked shares are summed.

---

## Protocol

For every pair `(i, j)` with `i < j`, a random 64-bit mask `r_ij` is generated by insurer `i` and delivered to insurer `j`. Each insurer then computes a local masked share:

```
s_A = x_A + r_AB + r_AC
s_B = x_B ‚àí r_AB + r_BC
s_C = x_C ‚àí r_AC ‚àí r_BC
```

Only `s_A`, `s_B`, `s_C` are sent to the aggregator. Every mask appears once with `+` and once with `‚àí`, so:

```
s_A + s_B + s_C = x_A + x_B + x_C    (all masks cancel)
average         = (s_A + s_B + s_C) / 3
```

The aggregator learns only the sum (and therefore the average). Individual claims remain private.

## Running it

Requires Python 3.7+ (uses only the stdlib).

```bash
python server.py
```

Then open each page in a **separate** browser tab or window:

| Role        | URL                                   |
| ----------- | ------------------------------------- |
| Home        | <http://127.0.0.1:8765/>              |
| Insurer A   | <http://127.0.0.1:8765/party/a>       |
| Insurer B   | <http://127.0.0.1:8765/party/b>       |
| Insurer C   | <http://127.0.0.1:8765/party/c>       |
| Aggregator  | <http://127.0.0.1:8765/aggregator>    |

Each insurer enters their claim and clicks *Start Protocol*. Once all three have submitted, the aggregator page reveals the average.

To start a fresh round, click **Reset session** on the home page.

### Server endpoints

- `POST /api/mask` ‚Äî an insurer relays a pairwise mask to its counterparty
- `GET  /api/masks?for=X` ‚Äî insurer `X` fetches incoming masks addressed to it
- `POST /api/share` ‚Äî an insurer submits its final masked share
- `GET  /api/result` ‚Äî aggregator retrieves masked shares and their sum (only once all three are submitted)
- `GET  /api/state` ‚Äî public status (which insurers have submitted so far)
- `POST /api/reset` ‚Äî clear all state for a new round

## Security notes

This is an educational demo, not production-grade:

- **Mask exchange is in plaintext through the server.** In a real deployment, each pair should exchange masks over an end-to-end encrypted channel so the coordinator cannot see them. The aggregator-facing endpoints (`/api/share`, `/api/result`) already only touch masked values.
- **No authentication.** Anyone who can reach the server can claim to be any insurer.
- **Fixed-point arithmetic** (√ó10‚Å∂) is used so decimals work with BigInt on the client. Pick a scale that fits your expected range.
- **Collusion.** As with any pairwise-masking scheme, two colluding insurers (or an insurer colluding with the aggregator) can reconstruct the third insurer's input ‚Äî this is inherent to 3-party additive secret sharing.
